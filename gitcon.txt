폴더 감시시작  환경 만들기 (초기화)
폴더안에들어가서 우클릭후 bash프로그램 실행 

git init 
(프로젝트 폴더안에서 관리김시를 하기위한 환경을 설정하기
 위한 것으로 최초로 한번만 실행)

git status
(현재 git프로그램이 감시하고있는 상태를 확인하기 위한 명령어)
→상태확인을 위해 시도때도 없이 실행해도 됨

git remote add origin 깃허브주소
(현재 나의 로컬 저장소와 원격저장소의 연결고리를 설정)

git remote -v
(현재 설정되어있는 원격저장소의 정보를 확인)

파일을 수정 한 뒤 원격서버로 업로드 하는 방법(순성중요!!)
[전제:원격연결고리가 설정0, 깃 초기화0]
1. git add . (변경이 이루어진 파일들을 특별감시구역staging area로 보내는 명령어)

2. git commit -m "변경이력을 간단히 확인할 메세지"
(staging area에서 감시당하던 파일들에게 일련번호를 생성하여 히스토리를 기록)

3.git push origin mater
(연결되어있는 원격저장소로 파일을 업로드)

*pull 안될때 git pull origin 브런치명 --allow-unrelated-histories 입력
